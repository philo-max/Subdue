**版本**：v0.2
**日期**：2026-06-19
**状态**：MVP 规划阶段（名称、托管策略、平台顺序已确定）

---

## 1. 产品概述

### 1.1 一句话定位
一个**无广告、隐私优先**的订阅管理工具，帮用户看清"钱到底花在哪儿"，并在涨价或自动续费前提醒他们。提供**开源自部署版**和**云端自动化商业版**两条路径。

### 1.2 要解决的问题
| 用户痛点 | 现状 | 我们的解法 |
|---|---|---|
| 不知道自己订了多少服务 | 散落在各个 App / 银行账单里 | 一个仪表盘统一展示 |
| 免费试用忘记取消被扣款 | 没有任何提前提醒 | 到期前 N 天主动推送 |
| 服务悄悄涨价没发现 | 银行扣款记录不会主动告诉你 | 金额变化自动检测+提醒 |
| 不信任要订阅类 App 偷偷收集数据来卖广告 | 市面同类产品大多靠广告/卖数据盈利 | 开源可审计代码 + 零广告承诺 + 可选自部署 |

### 1.3 产品原则（不可妥协的红线）
1. **零广告** — 任何版本，永久不接入广告 SDK。
2. **隐私优先** — 默认本地存储；云端同步功能必须显式授权且可随时导出/删除全部数据。
3. **开源核心** — 核心引擎（数据模型、计算逻辑、UI）MIT 协议开源，任何人可审计、可自部署。
4. **优雅降级** — 商业版断网/不订阅 Pro 时，应退回到开源版功能，不锁死已录入的数据。

## 3. 版本范围速览

> 详细技术拆分见此前的架构文档，这里只列产品边界。

**开源版（Free / Self-Host）**
手动录入 + 服务数据库自动补全、CSV 账单本地解析、到期/涨价提醒、本地仪表盘、本地加密存储、数据导入导出。

**商业版（Pro / Family，订阅制，无广告）**
开源版全部功能 + 邮件自动扫描、银行 API 直连、支付宝/微信账单解析、多设备云同步、家庭共享、AI 智能分析报告。

## 5. 核心功能详细规格

### 5.1 订阅管理（CRUD）
- 字段：服务名称、Logo、金额、币种、扣款周期（周/月/季/年/自定义天数）、首次扣款日、下次扣款日（自动计算）、分类、备注、状态（启用/已取消/暂停）。
- 取消订阅时**不删除记录**，仅标记为"已取消"，保留历史数据用于年度报告。

### 5.2 服务数据库与自动填充
- 内置约 2000 条常见订阅服务（流媒体、云存储、AI 工具、健身、新闻等），含官方 Logo、典型定价、典型周期。
- 用户输入名称时模糊匹配并提示选择；找不到则手动填写并可选择"贡献到社区数据库"（开源协作，PR 到 GitHub 仓库）。

### 5.3 仪表盘 / 统计
- 顶部卡片：本月支出、本年累计、预计年度总支出。
- 分类占比饼图（流媒体 / 软件 / 健身 / 云服务 / 其他）。
- 即将到期列表（按时间排序，7 天内高亮）。
- 年度回顾报告（类似"年度账单"，可生成分享卡片，无广告无水印）。

### 5.4 提醒通知
- 本地推送通知，默认提前 3 天，可在设置中调整（1/3/7/自定义天数）。
- 涨价提醒：检测到同一服务扣款金额上升时单独推送。
- 通知文案示例：「Netflix 将在 3 天后扣款 ¥35，点击查看详情」。

### 5.5 数据导入导出
- 导出：完整 JSON（含所有字段）+ 简化版 CSV（用于 Excel 查看）。
- 导入：支持自家 JSON 格式 + 通用银行 CSV 格式（提供模板说明）。
- 用途：换设备迁移、自部署用户备份、防止厂商锁定。

### 5.6 设置
- 主货币选择、是否启用生物识别锁、提醒提前天数、深色模式、语言、数据库加密密码管理。

## 6. 核心用户流程

**首次使用流程**
打开 App → 选择"开始使用"（无需注册，本地优先）→ 引导添加第一个订阅（提供常见模板快速选择：Netflix/Spotify/iCloud等）→ 看到仪表盘 → 引导开启通知权限。

**添加订阅流程**
点击"+"→ 搜索服务名称（自动补全）→ 确认/修改金额与周期 → 选择分类 → 保存 → 自动计算下次扣款日并加入提醒队列。

**CSV 导入流程（P1）**
设置 → 导入账单 → 选择文件 → 系统自动识别周期性扣款候选项并展示置信度 → 用户逐条确认/忽略 → 批量导入完成。

## 7. 数据模型（核心实体）

```
Subscription
├── id, name, logo_url
├── amount, currency
├── billing_cycle (weekly/monthly/quarterly/yearly/custom_days)
├── first_billed_at, next_billed_at (计算字段)
├── category_id, status (active/cancelled/paused)
├── notes, created_at, updated_at

Category
├── id, name, icon, color

ServiceCatalog（内置/社区数据库，只读）
├── id, name, logo_url, typical_price, typical_cycle, category_id

Reminder
├── id, subscription_id, remind_before_days, last_triggered_at

UserSettings
├── primary_currency, reminder_default_days, theme, locale, biometric_lock_enabled
```

## 8. 非功能性需求

| 维度 | 要求 |
|---|---|
| **隐私** | 开源版零遥测（不上报任何使用数据）；商业版遥测需 opt-in 且可随时关闭 |
| **安全** | 本地数据库强制加密；商业版邮件/银行授权 Token 仅本地或加密存储，不落地明文 |
| **性能** | 冷启动 < 2 秒；1000 条订阅记录下仪表盘渲染 < 500ms |
| **离线优先** | 开源版功能 100% 可离线使用 |
| **可访问性** | 支持系统字体缩放、屏幕阅读器基本兼容 |
| **多语言** | MVP 至少中文+英文 |
| **无广告承诺** | 写入 README 和 App 商店描述，作为品牌承诺公开 |

## 已确认决策

| 问题 | 决策 |
|---|---|
| **产品名称** | **Subdue**（搜索确认无同类应用同名，可用；寓意"驯服你的订阅"） |
| **开源版托管方式** | **提供官方免费托管的 Web 实例**（不只是发代码）。理由：目标用户里很多是"不会做产品/不懂自部署"的普通人，自部署门槛会挡住大部分人。代码依然 MIT 开源，技术用户仍可自行部署，两条路并存。 |
| **MVP 平台顺序** | **Web 优先**，验证最快，再扩展到移动端和 PC 端（Tauri 包装同一套前端）。 |

> 由于官方提供免费托管实例，开源版的"零遥测"承诺需要更精确表述：**官方实例不主动收集行为数据用于分析/广告，仅保留运行必需的最少日志**；自部署版本则完全由用户掌控、零回传。这一点需要写进隐私政策。
